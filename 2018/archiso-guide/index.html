Arch Linux 作为高度可定制化的 Linux 发行版，连制作镜像都提供了一系列顺手的工具。这里简要介绍一下 Archiso 的用法。

正文

准备工作

  • 已经安装好的 Arch Linux x86_64
  • 能联网的电脑
  • 安装好 archiso
  • 建立一个工作目录，以 ~/archiso/为例。
  • 洗把脸

接下来的操作都需要 root 权限（主要是为了避免文件所有权问题），所以……

$ su root

开始

Archiso 提供了几个模板，这里使用倾向于自定义的 releng

# cp -r /usr/share/archiso/configs/releng/* archlive

安装软件包

编辑 packages.both 设置需要安装的软件包。如有需要，可以编辑 packages.i686 或 packages.x86_64 以单独设置不同架构要安装的软件包

安装 multilib 仓库中的软件包（选）

如果你制作的 Archiso 只用于 x86_64，直接编辑 pacman.conf 启用就好

如果不是，那你需要两份分别用于 x86_64 和 i686 的 pacman 配置文件

# cp pacman.conf pacman-x86_64.conf && cp pacman.conf pacman-i686.conf

编辑 build.sh，将 (Line 249)：

run_once make_pacman_conf

# Do all stuff for each airootfs
for arch in i686 x86_64; do
    run_once make_basefs
    run_once make_packages
done

run_once make_packages_efi

for arch in i686 x86_64; do
    run_once make_setup_mkinitcpio
    run_once make_customize_airootfs
done

改为：

cp -v releng/pacman-x86_64.conf releng/pacman.conf
run_once make_pacman_conf

# Do all stuff for each airootfs
for arch in x86_64; do
    run_once make_basefs
    run_once make_packages
    run_once make_packages_efi
    run_once make_setup_mkinitcpio
    run_once make_customize_airootfs
done

echo make_pacman_conf i686
cp -v releng/pacman-i686.conf releng/pacman.conf
cp -v releng/pacman-i686.conf ${work_dir}/pacman.conf


for arch in i686; do
    run_once make_basefs
    run_once make_packages
    run_once make_packages_efi
    run_once make_setup_mkinitcpio
    run_once make_customize_airootfs
done

最后，编辑 pacman-x86_64.conf 启用 multilib 仓库

[multilib]
SigLevel = PackageRequired
Include = /etc/pacman.d/mirrorlist

添加文件

这一步一定要用 root 权限，以避免文件所有权问题！！！

把 archiso/airootfs 当成根目录就好，把你需要添加的文件 / 配置文件云云复制进去

注意：用户 home 文件夹的文件要添加到 archiso/airootfs/etc/skel 中

添加普通用户

以用户名 archman 为例，编辑 archiso/airootfs/root/customize_airootfs.sh，添加：

useradd -m -G wheel -s /bin/bash archman
passwd -d archman //这两行的作用是禁用用户密码
passwd -u archman

配置显示管理器（选）

以 SDDM 为例，创建软链接

# ln -s /usr/lib/systemd/system/sddm.service archiso/airootfs/etc/systemd/system/display-manager.service

如果需要显示管理器开机启动，则编辑 archiso/airootfs/root/customize_airootfs.sh，将：

systemctl set-default multi-user.target

改为：

systemctl set-default graphical.target

生成镜像

完成配置后，就生成镜像吧

# mkdir archiso/out/
# cd archiso
# ./build -v

使用 ./build -h 来看看还可以使用哪些参数
